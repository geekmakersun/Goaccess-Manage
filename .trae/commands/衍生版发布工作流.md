## 🎯 任务：衍生版发布工作流

> ⚠️ **适用场景**：此工作流仅适用于衍生仓库。主仓库请使用 `主仓库发布工作流`。
>
> 📌 **重要说明**：
> - ❌ 不修改 `README.md`
> - ❌ 不修改 `CHANGELOG.md`
> - ✅ 仅同步代码变更到衍生仓库

### 📋 步骤1：分析代码变更 🔍
- 📂 使用 `git status` 查看当前修改的文件
- 📝 使用 `git diff` 查看具体的代码变更内容
- 🧠 基于变更内容，AI自动分析并总结修改了什么

### 📋 步骤2：生成提交信息 ✍️
- 📅 使用 `date '+%Y-%m-%d %H:%M:%S'` 获取当前日期时间
- 🤖 AI根据代码变更自动生成提交标题，要求：
  - 🔖 包含变更类型（新增/修改/修复/重构/文档/样式 等）
  - 📝 简明扼要概括核心变更
  - 📅 在标题末尾附加日期时间，格式：`[YYYY-MM-DD HH:MM:SS]`
- 📄 AI根据代码变更自动生成提交正文，要求：
  - 📋 列出具体修改了哪些文件
  - 🔍 描述每个文件修改了什么内容
  - 🏷️ 按变更类型分类说明
- 💡 提交信息格式示例：
  ```
  🔧 同步配置文件更新 [2025-05-28 15:30:00]

  - 从上游同步了 nginx.conf 配置更新
  - 更新了日志分析脚本
  - 调整了定时任务配置
  ```

### 🚀 步骤3：使用www用户提交代码 💾
- ➕ 使用 `git add .` 或 `git add -A` 添加所有修改的文件
- 👤 使用以下命令以www用户身份提交：
  ```
  git commit --author="www <www@localhost>" -m "AI生成的提交标题" -m "AI生成的提交正文"
  ```
- ☁️ 首先获取当前分支名：
  ```
  git branch --show-current
  ```
- ☁️ 获取当前仓库名称（用于识别这是哪个衍生版）：
  ```
  basename $(git remote get-url origin 2>/dev/null || echo "unknown") .git
  ```
- ☁️ **只推送到当前衍生版的远程仓库**，以www用户身份执行：
  ```
  su - www -c "git push origin <当前分支名>"
  ```
- ⚠️ **重要**：衍生版只推送到自己的 origin 远程仓库，**不推送到其他远程仓库**（如 japan、singapore 等）
- 🎯 **设计理念**：主仓库负责更新底座并推送到所有衍生版，衍生版只负责推送自己的定制内容到自己的远程仓库
- 📌 **仓库角色说明**：
  - `origin` - 当前衍生版的远程仓库，只推送自己的变更
  - 其他远程仓库 - 由主仓库统一推送更新，衍生版不涉及

> 💡 **小贴士**：衍生版工作流仅同步代码，不污染原版版本管理！🔄
